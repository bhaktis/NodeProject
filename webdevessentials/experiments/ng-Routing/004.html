Angular Routing With controllers

About the experiment

The routing functionality added by this step is provided by angular in the ngRoute module, which is distributed separately from the core Angular framework. The below experiment we see how to add multiple views to a Single page application. Each of these views will be assigned a seperate controller which manage the data for the page.

Books Food Items

Description

Angular js has feature called RouteProvider using which you can add multiple views in an HTML page. For this, we have to configure the config of the mainApp and provide the template URl and the controller for click of the link.We also need to specify the default URL where the page will be directed incase there are no matched found. The page will be viewed in the div for which we have specified the ng-view directive. In addition, the controller is specified for each view. This controller will create and manage the data model for the respective views.

Code snippet

            app.config(['$routeProvider', function ($routeProvider) {
    $routeProvider.
        when('/books', {
            templateUrl: 'views/Books/book.html',
            controller: 'BookCtrl'
        }).
        when('/food', {
            templateUrl: 'views/Food/food.html',
            controller:'FoodCtrl'
        }).
    otherwise({
        redirectTo: 'view/books.html'
    });
}]);
